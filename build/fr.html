Air France KLM (Inetum) - KAAS

01/01/2025 - en cours (1 an et 2 mois)

Le projet s’appuie sur un système ETL orchestré par Control-M qui récupère et transforme des données issues de Teradata, avant de les stocker temporairement dans PostgreSQL.

  • Fonctionnalités principales: extraction, transformation et chargement de données, collecte en temps réel, diffusion via HiveMQ, tableaux de bord personnalisables.
  • 3 développeurs, 1 product owner.
  • Méthodologie Kanban avec suivi continu des tâches et priorisation dynamique.
  • Objectif :
    • Mise en place d’une API REST fournissant des KPIs métier critiques pour Air France KLM.
  • Contexte :
    • Données issues d’un référentiel Teradata, intégrées à travers un ETL automatisé par Control-M.
    • Stockage temporaire dans une base PostgreSQL pour traitement.
  • Réalisations techniques :
    • Implémentation de batchs Spring Batch pour orchestration ETL.
    • Intégration d’un module "real-time" collectant continuellement des données externes.
    • Mise en place de flux dans HiveMQ pour la diffusion en temps réel.
    • Développement d’une UI Angular pour visualisation et personnalisation des dashboards.
  • Architecture :
    • Utilisation de SpringBoot pour structurer l’API.
    • Base de données primaire PostgreSQL avec appui sur MongoDB pour certains stockages.
    • Déploiement sur Azure Kubernetes Service pour scalabilité et résilience.
  • Collaboration :
    • Équipe agile en mode Kanban, communication continue entre PO et développeurs.
  • Gestion des données :
    • Scripts Control-M assurant la planification et l’exécution sécurisée des batchs.
Java 21, SpringBoot, Spring Batch, Spring JDBC, Teradata, MongoDB, PostgresQL, Control-M, Angular, Azure Kubernetes Service, Hive MQ

    • Le flux est ensuite traité et restitué sous forme de contenu HTML.
